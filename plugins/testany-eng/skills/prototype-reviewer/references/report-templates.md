# 审查报告与准出证书模板

本文档定义 prototype-reviewer 的输出格式模板。

---

## 审查报告（未通过）

```markdown
# 原型审查报告

## 基本信息

| 项目 | 内容 |
|------|------|
| 沙箱目录 | [路径] |
| PRD 来源 | [路径] |
| User Journey 来源 | [路径] |
| 交付摘要 | [路径 / 缺失（P1）] |
| 变更基线 | [commit range / worktree + 归属确认] |
| 审查时间 | YYYY-MM-DD |
| 审查轮次 | 第 N 轮 |
| 审查结论 | **不通过** |

## 门一：上游对齐

### 需求覆盖表

| REQ-* | 需求描述 | Manifest 页面 | Journey 步骤 | 状态 | 说明 |
|-------|---------|-------------|-------------|------|------|
| REQ-01 | [描述] | [页面名] | [S1/S2] | ✅/⚠️/❌ | [未覆盖/部分覆盖原因] |

- Journey 步骤覆盖率：X / Y (Z%)
- 门一结论：[无 P0 可继续 / 存在 P0 阻塞]

## 门二：原型完整性

- P0 Journey Happy Path 可达性：[全通 / 断点列表]
- 状态矩阵覆盖率：已覆盖 M / 总数 T (Z%)
- 导航完整性：[全通 / 缺失列表]

## 门三：工程隔离

| 检查项 | 结果 |
|--------|------|
| 所有原型文件在沙箱内 | ✅/❌ |
| 变更基线已确定 | ✅ [commit range] / ✅ [归属确认] |
| 沙箱外零未授权变更 | ✅/❌ |
| 受控例外已记录 | ✅/❌/N/A |
| 原型路由在专属前缀下 | ✅/❌ |
| package.json 未修改 | ✅/❌ |
| 生产组件/页面/路由未修改 | ✅/❌ |

## 门四：下游可用性

- API Contract 输入评估：[具体 / 笼统 / 缺失]
- HLD 输入评估：[具体 / 笼统 / 缺失]
- 交付摘要与实际一致性：[一致 / 偏差]

## 问题清单

| # | 级别 | 门 | 问题描述 | 证据位置 | 建议修改 |
|---|------|-----|---------|---------|---------|
| 1 | P0 | Gate 3 | [描述] | [文件:行号] | [修改建议] |
| 2 | P1 | Gate 1 | [描述] | [PRD:REQ-XX] | [修改建议] |

## 放行决策

| P0 | P1 | P2 | 结论 |
|----|----|----|----|
| X 个 | Y 个 | Z 个 | **不通过** |

## 下一步

1. [按优先级列出需修复的问题]
2. 修复完成后执行 `/testany-eng:prototype-reviewer` 复审
```

## 准出证书（通过）

```markdown
# 原型准出证书

## 基本信息

| 项目 | 内容 |
|------|------|
| 沙箱目录 | [路径] |
| PRD 来源 | [路径] |
| User Journey 来源 | [路径] |
| 准出时间 | YYYY-MM-DD |
| 审查轮次 | 第 N 轮 |
| 审查结论 | **通过** |

## 上游对齐确认

- PRD 需求覆盖率：100%（本轮范围内）
- Journey 步骤覆盖率：X / Y (Z%)

## 原型完整性确认

- P0 Happy Path：全通
- 状态覆盖率：M / T (Z%)
- 导航完整性：全通

## 工程隔离确认

- 沙箱内零违规
- 零依赖新增
- 零未授权变更
- [受控例外：无 / 已验证（文件路径）]

## 下游可用性确认

- API Contract 输入：具体
- HLD 输入：具体

## 准出门槛确认

| P0 | P1 | P2 |
|----|----|----|
| 0 | 0 | ≤ 2 |

## 审查历程

| 轮次 | 日期 | P0 | P1 | P2 | 结论 |
|------|------|----|----|----|------|
| 1 | YYYY-MM-DD | X | Y | Z | 不通过 |
| 2 | YYYY-MM-DD | 0 | 0 | N | 通过 |

## 审查者

prototype-reviewer

## 准出确认

可以进入 API Contract / HLD 阶段：
- `/testany-eng:api-writer`
- `/testany-eng:hld-writer`

## 准出签章

`PASSED-{YYYYMMDD}-{沙箱目录名哈希前6位}`
```
