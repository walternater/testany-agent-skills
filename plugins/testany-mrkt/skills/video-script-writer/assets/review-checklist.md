# 视频脚本审核清单

## 使用说明
- ✅ 所有 Stage 完成后、进入制作前使用
- 逐项检查，在 [ ] 中打 ✓
- 严重问题必须修复后才能进入制作

---

## 一、一致性检查

- [ ] 脚本口播 ↔ 配音稿文案完全一致
- [ ] 脚本时间戳 ↔ 配音稿段落对齐
- [ ] 脚本画面描述 ↔ 录屏清单对应
- [ ] 脚本动画标注 ↔ Remotion 规格对应
- [ ] 脚本中文字幕 ↔ 配音稿字幕表一致
- [ ] Hook 方案 ↔ 脚本开头一致

## 二、时长检查

- [ ] YouTube 版总时长在 2:00–3:00 范围内
- [ ] 抖音版总时长在 0:45–1:30 范围内
- [ ] YouTube 版口播词数 260–450 词
- [ ] 抖音版口播词数 100–200 词
- [ ] Hook ≤ 5 秒
- [ ] 单段录屏 ≤ 20 秒

## 三、内容质量

- [ ] 痛点真实可信，不编造不夸大
- [ ] 产品演示中的功能在 TestAny 中真实存在
- [ ] CTA 明确，观众知道下一步做什么
- [ ] 英文口播自然口语化，无 marketing speak
- [ ] 中文字幕通顺，无翻译腔
- [ ] 未违反约束红线（客户名、真实密钥、贬低竞品）

## 四、制作可行性

- [ ] 录屏步骤描述足够详细，可按步执行
- [ ] TestAny 前置数据需求明确
- [ ] Remotion 组件规格可开发
- [ ] ElevenLabs 配音标注规范

## 五、品牌合规

- [ ] 语调符合品牌人设（专业但不冰冷）
- [ ] 无 "revolutionary" / "game-changing" 等 marketing speak
- [ ] 未贬低竞品，对比部分客观呈现
- [ ] Logo 使用规范
