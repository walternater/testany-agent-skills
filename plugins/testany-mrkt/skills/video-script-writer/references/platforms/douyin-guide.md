# 抖音 / TikTok 平台指南

## 平台特性

- **内容生命周期**：短爆发型，前 24–48 小时决定推荐量级
- **发现机制**：纯算法推荐，不依赖关注关系。完播率和互动率是核心指标
- **受众行为**：快速滑动浏览，注意力窗口极短（1.5–3 秒）
- **互动方式**：点赞、评论、收藏、转发

## 视频规格

| 规格 | 要求 |
|------|------|
| 分辨率 | 1080×1920（9:16 竖屏）或 1920×1080（16:9 横屏） |
| 时长 | 60–90 秒（本项目标准，从 YouTube 版精剪） |
| 帧率 | 30fps |
| 封面 | 第一帧或自定义封面，大字报式 |

**关于横屏 vs 竖屏**：
- 技术教程类内容用 **横屏（16:9）** 更合适——屏幕录制内容在竖屏上太小
- 抖音支持横屏视频，技术类创作者多用横屏
- 如果需要竖屏，则需要将录屏画面缩小放在上半部分，下半部分放字幕

## 内容策略

### 从 YouTube 版精剪的原则

抖音版不是简单地"把 YouTube 视频截短"，而是**重新结构化**：

```
YouTube（2:30）           →  抖音（1:00）
Hook: 5s                     Hook: 3s（更快更猛）
Pain: 35s                    Pain: 10s（保留最痛的一句）
Solution: 80s                Solution: 35s（只保留 Aha Moment）
CTA: 30s                     CTA: 12s（更直接）
```

### 精剪规则
1. Hook 要**更快**——抖音用户比 YouTube 用户更没耐心
2. Pain 部分只保留**最有共鸣的一句话**，不展开
3. Solution 只保留**最惊艳的操作**（Aha Moment），删除过渡和铺垫
4. CTA 更**直接**："Link in bio" 或 "评论区链接"

### 标题/文案
- 第一行就是钩子（和视频 Hook 一致或互补）
- ≤ 30 字符（避免折叠）
- 中文为主（配合中文字幕策略）

### 话题标签
- 4–6 个标签
- 混合：行业标签（#自动化测试 #DevOps）+ 流量标签（#程序员日常 #效率工具）
- 示例：`#自动化测试 #测试平台 #DevOps #程序员 #效率工具 #TestAny`

## 节奏指南

### 信息密度
- 每 8–10 秒一个新信息节点
- 60 秒视频 ≈ 6 个信息节点
- **绝对不能有任何"冷场"超过 5 秒**

### 完播率优化
- 前 3 秒是生死线——必须命中痛点或引发好奇
- 中间不能有"低谷"——每个段落结束时暗示"还有更厉害的"
- 结尾要有**信息回报**——让观众觉得"看完有收获"

### 字幕策略
- **大字幕**：关键痛点句子用大号字体，占屏幕中下部
- **关键词高亮**：痛点词、数字用不同颜色
- **字幕节奏**：跟口播同步，不要提前或滞后

## CTA 设计

- **不要**在视频中间加 CTA（抖音用户会认为是广告直接滑走）
- **结尾 CTA**：简短有力，"主页链接注册" / "评论区链接"
- **评论区**：置顶评论放注册链接
- **主页**：确保抖音主页有 testany.io 链接

## 互动引导

在视频结尾或评论区引导互动（提升算法推荐）：
- "你们团队最痛的测试问题是什么？评论区聊聊"
- "用过 Jenkins 的点赞，用过 TestAny 的扣 1"
- 回复每一条早期评论（触发算法二次推荐）
