# 目标受众画像

## 画像 A: 团队 QA / SDET — "永远追不上开发的人"

### 基本特征
- 职位：QA Engineer / SDET / QA Lead
- 团队规模：5–50 人的软件开发团队中
- 技术水平：会写测试脚本，熟悉 Postman/Playwright/Selenium

### 日常场景
- 开发一周推 15 个 PR，自己还在测上周的功能
- Postman collection 散在 5 个人的电脑上
- 凌晨 3 点被叫起来手动跑测试排查生产问题
- 有 Postman/Playwright/JMeter 三套工具，结果散落各处
- 老板问"测试覆盖率"答不上来
- 写了 200 个测试用例，手动跑一遍要 2 天

### 核心痛点
1. 测试跟不上开发迭代速度
2. 测试基础设施维护成本高（Jenkins 配置地狱）
3. 测试工具割裂，没有统一平台
4. 测试和 troubleshooting 完全脱节

### 情绪关键词
疲惫、无奈、被追赶、不被理解、加班

### 决策动机
统一管理测试资产、自动化触发执行、有可视化报告交差

### 在哪里出没
YouTube 技术频道、Reddit r/QualityAssurance、知乎技术话题、技术博客

---

## 画像 B: Solo Developer / 独立开发者 — "One Man Army"

### 基本特征
- 职位：独立开发者 / 初创团队 CTO / 全栈工程师
- 团队规模：1–3 人
- 技术水平：全栈，但测试经验有限

### 日常场景
- 一个人兼任 Dev / PM / DevOps / QA
- 知道该测试但永远排不上优先级
- 每次部署都是"deploy and pray"
- 有 Postman collection 但从来都是手动跑
- 搭 Jenkins 太重了，只想部署后自动跑个 smoke test

### 核心痛点
1. 没时间做测试——一个人干所有事
2. 搭建测试基础设施的门槛太高
3. 每次部署后没有自动化 smoke test
4. Katalon/Mabl 太贵，自建框架太耗时

### 情绪关键词
忙碌、孤独、侥幸、自嘲、务实

### 决策动机
最低成本建立基本测试覆盖，不拖慢一个人的交付节奏

### 在哪里出没
YouTube indie hacker 频道、抖音程序员日常、Twitter/X 开发者社区、Hacker News

---

## 两个画像的共性

- 都**知道测试很重要**，但都被其他事情挤占了优先级
- 都**已经有测试脚本**（至少有 Postman collection），只是没有自动化
- 都**不想从零搭建**——希望拿现有的东西直接用
- 都对 **AI 辅助** 持开放态度——如果 AI 能帮我写测试/跑测试，为什么不？
